# NAIS2-Forge 1.0.0

NAIS2-Forge의 첫 정식 릴리스입니다. 기존 NAIS2의 이미지 생성 흐름과 데이터 호환성을 유지하면서 캐릭터 프롬프트, 씬 일괄 생성, 이미지 관리, Cloudflare R2, EXIF 처리 및 스마트 툴을 확장했습니다.

## 주요 기능 추가

### 확장 캐릭터 프롬프트

- 한 캐릭터 카드에서 프롬프트와 네거티브 프롬프트를 탭으로 전환할 수 있습니다.
- 캐릭터 본체 프롬프트와 의상 프롬프트를 별도로 관리하고 각각 활성화할 수 있습니다.
- 한 캐릭터에 최대 5개의 버전을 쌓아 페이지처럼 전환할 수 있습니다.
- 화면에는 현재 버전의 이름만 표시하고 내부 식별자와 순번은 숨깁니다.
- 버전별 이름 변경과 개별 삭제를 지원합니다.
- 구버전으로 내보낼 때 각 버전을 독립 캐릭터로 변환하며 기존 데이터 구조와의 호환성을 유지합니다.
- 의상 구분 정보는 프롬프트 내 주석 표식을 사용해 구버전에서도 프롬프트 손실 없이 읽을 수 있습니다.

### 씬 모드 확장

- 선택한 캐릭터들을 원하는 순서로 큐에 넣고, 각 캐릭터마다 전체 씬 생성을 반복할 수 있습니다.
- 씬별로 별도 캐릭터를 추가할 수 있습니다.
- 씬마다 캐릭터 버전 번호를 지정할 수 있으며, 해당 버전이 없으면 마지막 버전을 사용합니다.
- 씬마다 의상 프롬프트 사용 여부를 지정할 수 있습니다.
- 캐릭터 패널, 캐릭터 큐, 동적 씬 설정 사이의 활성화 상태를 동기화했습니다.
- 가로, 세로에 이어 정사각형 썸네일 표시 모드를 추가했습니다.
- 씬 선택, 전체 선택 및 해제, 삭제, 내보내기 등 기존 일괄 작업 흐름을 유지합니다.

### 이미지 레퍼런스 및 바이브 관리

- 메인 화면에 레퍼런스 이미지와 바이브 트랜스퍼 전용 사이드 패널을 추가했습니다.
- 파일을 패널에 드래그해 바로 등록할 수 있습니다.
- 항목별 이름 변경, 활성화, 타입 및 강도 설정을 지원합니다.
- 레퍼런스와 바이브의 상호 배타 활성화를 양방향으로 처리합니다.
- 384:264 비율의 썸네일을 사용하고 원본은 API 전송이 필요한 시점에만 읽습니다.
- 썸네일 캐시를 제한해 다수의 원본 이미지가 메모리에 계속 남지 않도록 했습니다.

### Cloudflare R2

- S3 호환 API를 이용한 R2 파일 관리자 페이지를 추가했습니다.
- 버킷의 폴더 탐색, 폴더 생성, 파일 업로드 및 삭제를 지원합니다.
- 폴더 전용, 이름 중심, 이미지 미리보기의 세 가지 표시 방식을 제공합니다.
- 씬에서 ZIP 내보내기와 동일한 이미지 선정 규칙으로 결과 이미지를 R2에 직접 업로드할 수 있습니다.
- 메인 화면과 히스토리의 빠른 작업에서도 현재 이미지를 직접 업로드할 수 있습니다.
- R2 관리자 페이지 활성화 여부와 직접 업로드 기능을 분리했습니다.
- 폴더 삭제 시 내부 파일까지 처리하고 빈 폴더 생성 요청을 S3 규격에 맞게 수정했습니다.
- 선택적으로 EXIF를 제거한 결과를 업로드할 수 있습니다.

### EXIF 관리자

- 캔버스 재인코딩 방식으로 이미지 메타데이터를 제거합니다.
- PNG, JPEG 등 출력 형식을 숙련자 옵션에서 선택할 수 있습니다.
- 전용 관리 페이지, 페이지로 보내기, 즉시 제거 후 저장의 작업 방식을 제공합니다.
- 자동 저장 이름과 경로를 지정할 수 있습니다.
- 같은 이름이 존재하면 덮어쓰지 않고 번호를 붙여 저장합니다.
- 이름 탐색 결과를 가볍게 캐시해 동일 이름 파일이 많을 때의 저장 지연을 줄였습니다.

### 스마트 툴

- I2I, 인페인트, 업스케일, 배경 제거, 라인아트, 스케치, 색칠, 표정 변경, 이미지 정리 및 모자이크를 한 패널에서 사용할 수 있습니다.
- 도구별 설명과 예상 Anlas 비용을 이미지 해상도 및 계정 등급에 따라 표시합니다.
- 유료 도구 구역을 시각적으로 구분하고 카드 전체를 실행 버튼으로 사용할 수 있게 정리했습니다.
- 히스토리 이미지에서 I2I, 인페인트 및 레퍼런스로 보내는 경로를 수정했습니다.
- 인페인트 창을 닫을 때 마스크 편집 기록을 정리하고, 작업 중 이미지 교체 시에는 마스크를 유지합니다.
- 더 이상 사용하지 않는 Kaleidoscope 관련 도구와 로직을 제거했습니다.

### 라이브러리

- 기존 이미지 스택에 다른 이미지를 드래그해 바로 추가할 수 있습니다.
- 스택을 해제하지 않고 내부 이미지를 드래그해 순서를 변경할 수 있습니다.
- 스택 이동 중 기존 이미지 데이터와 순서를 유지합니다.

## 인터페이스 변경

- 주요 탐색 탭을 타이틀바 영역으로 이동해 작업 공간을 넓혔습니다.
- 좌측 패널과 프롬프트 패널의 너비를 마우스로 조절할 수 있습니다.
- 프롬프트 입력창 우하단에 긍정 및 부정 토큰 수를 표시합니다.
- 최대 512토큰을 넘으면 토큰 표시를 붉게 바꿔 경고합니다.
- 토큰 표시가 캐릭터 및 레퍼런스 팝업 위를 가리지 않도록 레이어 순서를 수정했습니다.
- 메인 이미지의 해상도 및 시드 정보는 평소 낮은 불투명도로 표시하고, 가리키면 선명하게 표시합니다.
- 표시된 시드를 적용하면 시드 잠금도 함께 활성화됩니다.
- 생성 취소 버튼을 눌렀을 때 새 시드가 만들어지던 문제를 수정했습니다.
- 메인, 히스토리, 씬 및 라이브러리의 이미지 빠른 작업 순서를 통일했습니다.
- 씬 도구 버튼의 순서와 JSON/ZIP 아이콘을 정리했습니다.
- 기본 프리셋 이름 변경을 허용하고, 프리셋이 하나뿐이면 삭제 버튼을 숨깁니다.
- 프리셋 선택 버튼과 목록 폭을 고정하고 긴 이름은 말줄임표로 표시합니다.
- 스마트 툴의 불필요한 헤더 공간을 제거했습니다.

## 성능 및 안정성

- 26MB 규모의 태그 검색 데이터를 Web Worker에서 지연 로딩해 시작 번들에서 분리했습니다.
- 주요 페이지와 프롬프트 생성기를 필요할 때만 불러오도록 코드 분할했습니다.
- 사용하지 않던 Transformers 및 PrismJS 의존성을 제거했습니다.
- 히스토리는 생성 및 삭제 이벤트로 즉시 갱신하며 매 생성마다 전체 폴더를 다시 읽지 않습니다.
- 히스토리 파일 수와 썸네일 캐시 수를 제한해 장시간 사용 시 메모리 증가를 줄였습니다.
- 스마트 툴, 이미지 참조 및 히스토리 패널이 필요한 전역 상태만 구독하도록 변경했습니다.
- 창 크기 변경 이벤트를 프레임 단위로 제한했습니다.
- 이미지 패널 타이머와 시스템 테마 리스너의 정리 처리를 보강했습니다.
- 불필요한 이미지 Base64 복사와 장기 보관을 줄였습니다.

## 제거 및 분리

- 원 개발자의 서버와 Discord에 의존하던 마켓 UI 및 연결 로직을 제거했습니다.
- NAIS3 이전 안내 팝업과 설정 페이지 배너를 제거했습니다.
- 테스트 후 불필요하다고 확인된 별도 공식 웹 페이로드 생성 옵션을 제거했습니다.
- Kaleidoscope 스타일 스마트 툴을 제거했습니다.
- 앱 저장소 식별자를 Forge 전용으로 분리해 기존 정식 NAIS2 데이터와 설정을 공유하지 않습니다.

## 데이터 및 호환성

- 기존 NAIS2 프롬프트, 프리셋, 캐릭터 및 장면 데이터 구조를 가능한 범위에서 유지합니다.
- 이미지에 저장되는 `promptParts`는 사용자 프롬프트를 분석해 강제로 나누는 기능이 아닙니다. 생성 당시의 입력 칸 구성을 PNG의 전용 메타데이터에 저장하고 다시 불러올 때만 복원합니다.
- `promptParts` 정보가 없는 기존 이미지는 임의 문자열 규칙으로 자동 삼분할하지 않습니다.
- API 키, R2 인증 정보, 사용자 데이터, 생성 이미지 및 업데이트 서명 개인키는 저장소에 포함하지 않습니다.

## 업데이트 및 배포

- 앱 버전을 Forge `1.0.0`으로 재정립했습니다.
- 업데이트 확인 대상은 `IZTACIYU/NAIS2-Forge`의 GitHub Releases입니다.
- 이후 업데이트에는 서명된 Windows 설치 파일과 `latest.json`이 함께 제공됩니다.

## 알려진 사항

- 첫 Rust/Tauri 빌드는 의존성 컴파일 때문에 오래 걸리며 `src-tauri/target`이 10GB 이상 커질 수 있습니다. 이는 개발용 빌드 캐시이며 설치된 프로그램 용량이 아닙니다.
- `src-tauri/target`은 프로그램과 개발 서버를 종료한 뒤 삭제해도 사용자 데이터에 영향을 주지 않습니다.
- 태그 검색 Worker 파일은 태그 데이터가 포함되어 크지만 해당 기능을 사용할 때만 로드됩니다.
